# Contributing

made this fast, tested hard, and there are definitely edges. appreciate any help.

## how to contribute

1. fork the repo
2. make your changes
3. write tests if you're adding features
4. make sure tests pass: `npm test`
5. submit a pr

## code style

just keep it readable. i'm not picky.

## testing

```bash
npm test
npm run benchmark
```

make sure these pass before submitting.

## questions

open an issue or start a github discussion
